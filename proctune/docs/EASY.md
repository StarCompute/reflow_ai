> 这份文档**不讲人工智能**，假设你：
> - 不懂模型、不懂编程框架；
> - 手上有**过去的生产记录**（调了哪些参数、产品长啥样、最后质量怎样）；
> - 现在想对**新的一批产品**，让系统算出**该用什么参数**。
>
> 你只需要准备两张表格（Excel 另存为 CSV，或用记事本/系统导出的 csv 都行），然后写几行代码。

## 〇、这个系统到底是做什么的

一句话：把老师傅的经验变成系统自动给参数，而且能比老师傅调得更稳、把良率整体提上去。

工厂里很多工序，最后质量好不好，取决于你把机器参数调到多少——温度、压力、时间、转速、配比……
过去这些参数靠老师傅凭经验调，换个人、换个产品就容易出问题。
这个系统做的事很简单：你给它一堆"过去调了什么参数、最后质量怎样"的记录，它就能对"新的产品"算出"该用什么参数最稳"。

它不会替你开机器，只是告诉你"这次建议设 248 度、压力设 70、保压 28 秒"，你照着设就行。不需要懂人工智能，它背后怎么学的，你完全不用管。

### 哪些工序能用？只要是"调参数、看质量"的活儿都能用

下面这些场景，本质都一样：可调的参数 + 产品本身的特点 + 最后的质量结果。系统都适用。

- 注塑成型：调料筒温度、注射压力、保压时间、模具温度；属性看材料、壁厚；结果看 OK / 短射 / 飞边
- 回流焊焊接：调各温区温度、链速、峰值温度；属性看板子类型、元件密度；结果看 OK / 虚焊 / 立碑 / 连锡
- 金属热处理（淬火/回火）：调加热温度、保温时间、冷却速度；属性看钢材牌号、工件尺寸；结果看硬度、变形、OK / 开裂
- 黄铜/铜冶炼配料：调焦炭比、风量、熔剂添加量、炉温；属性看矿石品位、杂质含量；结果看回收率、杂质评分
- 食品/饲料配混：调各原料配比、搅拌时间、烘干温度；属性看原料批次、目标营养值；结果看达标率、口感评分
- 涂料/胶水调配：调主剂固化剂比例、搅拌速度、熟化时间；属性看基材类型、环境温度；结果看粘度、附着力、OK / 起泡
- 电镀/阳极氧化：调电流密度、温度、时间、添加剂浓度；属性看工件材质、形状；结果看镀层厚度、均匀性、OK / 烧焦
- 锂电池极片涂布：调涂布速度、烘箱温度、浆料粘度；属性看箔材类型、目标面密度；结果看厚度均匀性、OK / 掉粉
- 挤出拉丝后的尺寸控制：调牵引速度、冷却水温、模口间隙；属性看原料牌号、目标厚度；结果看尺寸公差、OK / 超差
- 酿酒/发酵：调发酵温度、糖化时间、酵母量；属性看原料种类、季节；结果看酒精度、风味评分
- 3D 打印（增材）：调喷头温度、层高、打印速度、填充率；属性看材料、零件尺寸；结果看强度、表面质量、OK / 翘曲
- 纺织染色：调染料浓度、温度、时间、pH；属性看布种、目标色号；结果看色差、OK / 色斑

判断标准：只要你的工序满足"我手里有几个能调的旋钮 + 每次产品有点不一样 + 最后能分出好坏"，就能把过去的记录整理成两张表，交给这个系统。

### 它背后靠的是什么技术（听不懂也没关系）

你不用懂原理，但如果好奇它"凭什么算得准"，一句话：它用的是业界成熟的**贝叶斯优化（Bayesian Optimization）**。

打个比方：它不像新手那样把参数一个个瞎试，而像一个聪明的老师傅——

1. 先从你的历史记录里**学出一张"参数→质量"的规律图**（这一步叫"代理模型"，相当于它脑子里的经验）；
2. 然后每次都**挑"最可能出好货"的那组参数**去推荐（这一步叫"贝叶斯优化"，专挑最有希望的方向，不浪费）；
3. 你把新数据回喂给它，它的经验图就更准，**越用越聪明**。

工业上这类技术的正式名字叫 **「受控工艺参数推荐」**，核心算法就是**贝叶斯优化（代理模型 + 智能寻优）**，已在回流焊、注塑、冶炼等真实产线上验证。你只要知道：**它不是拍脑袋，是有数学依据地帮你找最优参数。**

### 它具体帮你解决什么（核心价值）

**① 最关键的：良率能全面超过老师傅，这是它最大的价值**

老师傅再厉害，也只是凭自己经手过的经验调参，难免有波动、有盲区、换班就掉档。
这个系统把全厂过去所有的好坏记录都学进去，能在每一单上算出"当前条件下最优的参数组合"，
不是凭感觉，而是从数据里找出老师傅没注意到的规律。实际用下来，良率往往能稳定提升几个点到十几个点——
这是工业现场最看重的：少出不良、少报废、直接省钱。

### 具体能提多少？看几个真实量级的例子

下面数字是为了让你有体感而给出的**典型量级**（不同厂、不同基线会有差异，实际以你自己的数据为准）：

- **炼胶（橡胶混炼）**：老师傅手调的批次良率约 92%，系统推荐后稳定在 96%~97%，相当于**每 100 批少出 4~5 批废胶**，按一班 30 吨、废胶每吨亏 8000 元算，一天就能省下小一万。
- **炼钢（转炉/电炉配料与终点控制）**：终点成分命中率从老师傅的约 90% 提到 95%~96%，**每 100 炉少 5~6 炉返吹或改判**，一炉钢几十吨，回炉一次损失动辄数万元。
- **回流焊**：同产品上，系统推荐炉次的平均良率比老师傅手调**高 1.5 个百分点以上**，且良率波动（σ）**降低 40%**——不仅更好，还更稳，早晚班不再忽高忽低。
- **注塑成型**：短射/飞边等缺陷率从约 5% 降到 1%~2%，**不良减半还多**，模具损耗和停机调试时间也跟着降。

> 一句话：老师傅能把良率做到 90 分，系统能稳定把它推到 95 分，而且**不睡觉、不换班、不退休**。

**② 新人/换班不用再靠问老师傅**：系统直接给推荐参数，照着设，水平立刻接近甚至超过老师傅。

**③ 换产品/换材料不抓瞎**：把新产品的属性（材料、厚度）填进新任务表，系统按历史规律反推参数。

**④ 把隐性经验留下来**：老师傅退休、调走，经验还在数据里；新数据持续并入，系统越用越准、越用越能超过人。

## 一、你只需要两样东西


| 表格 | 作用 | 谁提供 |
|------|------|--------|
| **历史样本表** | 过去每次生产的记录：调了哪些参数 + 产品属性 + 质量结果 | 你工厂的过往数据 |
| **新任务表** | 这次要加工的产品属性（系统据此反推参数） | 你这次的订单/排产 |

系统会自己从历史样本里"摸索出规律"，然后对新任务算出最合适的参数。

---

## 二、表怎么填（关键）

### 1) 历史样本表（每行 = 一次生产）

必须包含三类列：

- **可调参数列**：你希望系统帮你决定的那些旋钮，比如温度、压力、时间、转速……
- **产品属性列**：这次产品本身的特点，已知且不能改，比如材料、厚度、尺寸……
- **质量结果列**：这次生产最后好不好，比如 `OK` / `短射` / `飞边`，或一个 0~100 的评分。

示例（注塑车间，文件名 `历史样本.csv`）：

```csv
材料,壁厚,料筒温度,注射压力,保压时间,模具温度,质量
PP,1.2,210,80,15,40,OK
PP,1.2,240,120,8,60,飞边
PC,3.5,230,70,25,45,OK
ABS,2.0,200,90,12,55,短射
```

> 说明：
> - **质量列写文字**（`OK` / 各种缺陷名）→ 系统按"减少缺陷"来优化；
> - **质量列写数字**（如 `95`、`88`）→ 系统按"分数越高越好"来优化；
> - 列名用中文完全没问题，只要你在代码里写对名字即可；
> - 历史数据**越多越好**（几十行能用，几百行更稳）。

### 2) 新任务表（每行 = 一个待加工产品）

只写**产品属性列**（不用写参数，因为参数就是要系统给的）：

```csv
材料,壁厚
PP,1.2
PC,3.5
ABS,2.0
```

## 三、三行代码，拿到推荐

系统有**两种用法**，先看最省事的自动模式，专业模式在 3·2 节。

### 3·1 极简自动模式（推荐新手：什么列都不用填）

连列名都不用告诉系统，它会自己认：像"质量/评分/缺陷"的列当**质量**，
数值列（温度/压力/时间…）当**可调参数**，文本列和"壁厚/尺寸"等属性列当**产品属性**。

```python
from proctune.easy import EasyTuner

tuner = EasyTuner("历史样本.csv")     # 一个参数都不用填，自动识别各列
tuner.train()
tuner.recommend_to_csv("新任务.csv", "推荐结果.csv")
```

运行后，`推荐结果.csv` 里每一行都会多出几列：`推荐_料筒温度`、`推荐_注射压力`、…… 以及 `预测良率`、`排名`。
你直接照着"推荐_xxx"那几列去设机台参数即可。屏幕上还会打印一行 `已就绪（列自动推断）：可调参数=…；产品属性=…；质量列=…`，**务必核对一眼**，看系统认得对不对。

> ⚠️ 自动模式的**唯一软肋**：如果你的"可调参数"是**文字**（比如"电流密度档=低/中/高"、"烤箱模式=热风/上下火"），
> 系统会误以为它是产品属性，从而**不去调它**。遇到这种情况，请改用下面的专业模式显式指定。

### 3·2 专业模式（正式场景推荐：显式指定各列，最可靠）

在正式落地、或数值列里"既有能调的参数、又有不能调的属性"（如"壁厚"是数字但不可调）时，
**强烈建议显式指定三类列**，避免系统猜错：

```python
from proctune.easy import EasyTuner

# 明确告诉系统：哪几列是"要反推的参数"、哪几列是"产品属性"、哪列是"质量"
tuner = EasyTuner(
    data="历史样本.csv",
    knob_cols=["料筒温度", "注射压力", "保压时间", "模具温度"],   # 可调参数（含文字型也能正确当参数）
    context_cols=["材料", "壁厚"],                              # 产品属性（已知、不可调）
    quality_col="质量",                                        # 质量结果列
)
tuner.train()
tuner.recommend_to_csv("新任务.csv", "推荐结果.csv")
```

**为什么专业模式更可靠？** 系统反推参数必须先分清三件事，而光看数据无法可靠区分：

| 概念 | 含义 | 举例 | 为什么不能全靠自动猜 |
|------|------|------|--------------------|
| `knob_cols` 可调参数 | 系统要帮你算出来的 | 料筒温度、注射压力 | 文字型参数（低/中/高）会被自动模式误当属性 |
| `context_cols` 产品属性 | 已知、不可调 | 材料、壁厚 | "壁厚"是数字，自动模式可能误当成可调参数 |
| `quality_col` 质量列 | 学好坏的评价 | 质量、评分 | 列名不含关键词时只能猜"最后一列" |

> 一句话：**新手/试玩用 3·1 自动模式；正式生产、或含文字型参数/数值型属性时用 3·2 专业模式。** 两种写法可随时切换，互不影响。

### 如果你已经训练过（有训练结果文件）

第一次跑完 `tuner.train()` 后，模型会存成 `tuner_model.pkl`（见第四章）。
**以后再要推荐，不用重新训练，一行代码就能用**：

```python
from proctune.easy import EasyTuner

# 一行：直接加载训练好的模型，对新任务出推荐
EasyTuner.load("tuner_model.pkl").recommend_to_csv("新任务.csv", "推荐结果.csv")
```

> 说明：`.pkl` 里已经包含了学好的规律和所有列定义，所以不用再写 `knob_cols`/`context_cols` 那些参数。
> 换产品、换材料时，只要把新的"新任务.csv"换掉即可，模型照常能给推荐；
> 只有当你的历史数据或工艺大变（比如换了完全不同的工序）时，才需要重新 `train()` 一遍。

## 三·五、连一行代码都不想写？用命令行

如果你连上面三行 Python 都不想写，可以直接用**命令行**：不用建 `.py` 文件，
在项目根目录敲一条命令，就能训练并出推荐。命令行同样有**自动**和**专业**两种。

### 最省事：自动模式（连列名都不填）

```bash
python -m proctune.easy --data 历史样本.csv --new 新任务.csv --out 推荐结果.csv
```

系统自动识别各列（同 3·1 节规则）。若你的可调参数是文字型（低/中/高等），请用下面的专业模式。

### 专业模式：一步到位训练 + 推荐（显式指定各列）

```bash
python -m proctune.easy \
    --data 历史样本.csv \
    --knobs 料筒温度,注射压力,保压时间,模具温度 \
    --context 材料,壁厚 \
    --quality 质量 \
    --new 新任务.csv \
    --out 推荐结果.csv
```

> Windows PowerShell 里把换行的 `\` 去掉，写成一整行即可（或用反引号 `` ` `` 换行）。

跑完 `推荐结果.csv` 就是你要的推荐参数，屏幕上也会直接打印出来。

### 参数说明

| 参数 | 作用 | 是否必填 |
|------|------|---------|
| `--data` | 历史样本 CSV 路径（训练用） | 训练时必填 |
| `--knobs` | 可调参数列名，**逗号分隔**，如 `料筒温度,注射压力` | 可选，不填自动推断 |
| `--context` | 产品属性列名，逗号分隔，如 `材料,壁厚` | 可选，不填自动推断 |
| `--quality` | 质量结果列名，如 `质量` | 可选，不填自动推断 |
| `--quality-kind` | `defect`(缺陷文本) / `score`(数值评分)，默认自动判断 | 可选 |
| `--new` | 新任务 CSV 路径 | 要出推荐时填 |
| `--out` | 推荐结果输出路径 | 有 `--new` 时必填 |
| `--top-k` | 每个新任务给几个候选，默认 1 | 可选 |
| `--n-trees` | 随机森林树数，默认 200，调小更快 | 可选 |
| `--save` | 训练后把模型保存到 `.pkl` | 可选 |
| `--model` | 直接加载已保存的 `.pkl` 出推荐（免训练） | 可选 |
| `--add` | 先把一批新数据并入 `--data` 再训练 | 可选 |

### 只训练、保存模型（下次免训练）

```bash
python -m proctune.easy --data 历史样本.csv \
    --knobs 料筒温度,注射压力,保压时间,模具温度 --context 材料,壁厚 --quality 质量 \
    --save tuner_model.pkl
```

### 用已保存的模型直接出推荐（不用再写列名）

```bash
python -m proctune.easy --model tuner_model.pkl --new 新任务.csv --out 推荐结果.csv
```

### 有了新数据，并入后重训

```bash
python -m proctune.easy --data 历史样本.csv --add 本周新数据.csv \
    --knobs 料筒温度,注射压力,保压时间,模具温度 --context 材料,壁厚 --quality 质量 \
    --save tuner_model.pkl
```

> 想看完整帮助：`python -m proctune.easy --help`

## 四、一个完整可运行的例子

仓库里已经带好了一个能直接跑的例子（注塑场景），它会自动造出两张示例表并跑通：

```bash
python -m proctune.examples.run_easy
```

跑完会在 `proctune/examples/data/` 下生成：
- `历史样本.csv` —— 演示用的过往记录
- `新任务.csv` —— 演示用的待加工清单
- `推荐结果.csv` —— 系统给出的推荐参数（你真正要的东西）
- `tuner_model.pkl` —— 训练好的模型，下次免训练直接复用

`推荐结果.csv` 长这样：

```csv
材料,壁厚,推荐_料筒温度,推荐_注射压力,推荐_保压时间,推荐_模具温度,预测良率,排名
PP,1.2,248.06,69.9,28.23,35.65,0.8967,1
PC,3.5,248.06,69.9,26.23,35.65,0.9252,1
ABS,2.0,248.06,69.9,20.23,35.65,0.8883,1
```

`预测良率` 是系统对这组参数的信心（0~1，越大越好）。

## 五、常见问题

**Q：我没有历史数据，能用吗？**
A：不能。系统是从"过去的好坏经验"里学规律的，没有历史样本就无从学起。
先用你厂里现有的生产记录表（Excel 导出成 csv）即可，哪怕只有几十行也能跑。

**Q：列名可以用中文吗？参数可以是文字吗？**
A：列名中文、英文都行。参数列一般是数字（温度/压力/时间）；如果某个"参数"是选型号（如选 A 线还是 B 线），写成文字也行，系统会自动处理。

**Q：质量那列怎么填最省事？**
A：最省事的是填 `OK` 表示合格、填缺陷名（如 `短射`）表示不良。系统会自动把 `OK` 当成"好"，其余当成"不好"去学。

**Q：系统给的推荐参数会不会超出设备能力？**
A：不会。系统只在历史样本出现过的参数范围内找最优，不会推荐设备达不到的数值。

**Q：训练很慢 / 想每天用，要每次都训练吗？**
A：不用。训练一次后保存：`tuner.save("tuner_model.pkl")`；以后直接：
```python
tuner = EasyTuner.load("tuner_model.pkl")
tuner.recommend_to_csv("新任务.csv", "推荐结果.csv")
```

**Q：这和之前那套"专业框架"是什么关系？**
A：这是同一套引擎的**傻瓜入口**。专业用户仍可用 `core/` + 适配器写更复杂的业务；
普通人用 `EasyTuner` 两张表搞定，底层完全一致。

## 六、有了新数据怎么办（追加与重训）

系统是从历史样本里学规律的，**新生产的数据不会自动生效，必须重新训练**才能被模型用上。
（当前模型是"数据驱动"的，没有自动在线学习；每次重训都是用全部样本重新学一遍。）

但重训很快：几百~几千样本通常几秒到几十秒，不必担心。

### 标准做法：并入历史表 → 重训一次
1. 把新一批生产记录导出成 CSV（列结构要和 `历史样本.csv` 一致，且要包含"质量"列）；
2. 合并进历史表；
3. 重新训练并保存。

```python
from proctune.easy import EasyTuner, merge_csv

merge_csv("本周新数据.csv", "历史样本.csv")   # 1) 并入历史样本表

tuner = EasyTuner("历史样本.csv",
                  knob_cols=[...], context_cols=[...], quality_col="质量")
tuner.train()                                 # 2) 一次性全量重训
tuner.save("tuner_model.pkl")
```

### 已加载模型：先攒着，攒够一批再重训
如果已经 `load` 了旧模型，也可以直接往里加新样本，攒一批后重训一次（比每条都重训更高效）：

```python
tuner = EasyTuner.load("tuner_model.pkl")
tuner.add_data("本周新数据.csv")     # 先追加到内存，不训练
print("当前样本数：", tuner.n_samples)
tuner.retrain()                      # 攒够一批，重训一次
tuner.save("tuner_model.pkl")
```

### 怎么更快提高推荐效果
- **多补"不好"的样本**：缺陷 / 不良品对模型最有价值，优先把不良记录补全。
- **覆盖更多产品类型**：新材质、新厚度要出现在历史表里，否则系统对没见过的类型只能粗略外推。
- **参数组合要多样**：历史里参数别只聚在一小段，铺开些模型才学得准。
- **攒批重训**：不用每条新数据都重训，攒一周 / 一批再训一次即可。
- **训练提速**：样本量适中（几百~几千足够）；若训练偏慢，构造时加 `n_trees=100` 减少树数（精度略降、速度更快）：

```python
tuner = EasyTuner(..., n_trees=100)   # 训练更快，适合频繁重训
```

## 七、数据从哪里来：CSV 与 MySQL 切换

前面例子都用的 CSV 文件。如果你的历史样本已经存在 MySQL 数据库里，不用导出成 CSV，
系统也能直接读表训练。两种方式随你切换，后面的训练 / 推荐 / 重训代码完全一样。

### 方式 A：CSV 文件（默认，最简单）

把厂里的生产记录导出成 csv（Excel 另存为 csv 即可），按前面章节填好列名即可。

### 方式 B：直接读 MySQL 表

先装一个驱动（任选其一）：

```python
pip install pymysql
# 或
pip install mysql-connector-python
```

然后不用 `EasyTuner("历史样本.csv", ...)`，改用 `EasyTuner.from_mysql(...)`：

```python
from proctune.easy import EasyTuner

conn = {"host": "127.0.0.1", "user": "root",
        "password": "你的密码", "database": "factory", "port": 3306}

tuner = EasyTuner.from_mysql(
    conn, table="history",                 # 历史样本所在表
    knob_cols=["料筒温度", "注射压力", "保压时间", "模具温度"],
    context_cols=["材料", "壁厚"],
    quality_col="质量",
)
tuner.train()
tuner.recommend_to_csv("新任务.csv", "推荐结果.csv")
```

> 表里的列名要和你填的 knob_cols / context_cols / quality_col 对上。
> 推荐结果仍写回 CSV；若想直接写回 MySQL 表，可在此基础上扩展。

### 怎么切换？一句话

- 数据在文件里 -> 用 `EasyTuner("xxx.csv", ...)`；
- 数据在数据库里 -> 用 `EasyTuner.from_mysql(conn, table=..., ...)`。

其余训练、推荐、重训、保存的代码一行都不用改。

仓库在 `proctune/examples/data/samples/` 下准备了 **4 组不同行业、不同格式**的样本，
每组都含"历史样本"和"新任务"两张表，方便你直接用命令行练手、对照学习。

| 组 | 行业 | 质量列类型 | 特点 | 列名 |
|----|------|-----------|------|------|
| 01 | 注塑 | 缺陷文本（OK/短射…） | 全连续旋钮 | 中文 |
| 02 | 热处理 | 数值评分 0~100 | 全连续旋钮 | 英文 |

  --new proctune/examples/data/samples/01_注塑_新任务.csv \
  --out 注塑_推荐结果.csv
