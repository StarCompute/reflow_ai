1. 概述与设计原则

要解决的问题：回流焊炉有 7~10 个独立温区、链速、风量等多维设定参数。一条 3 温区×2 梯度×链速的搜索空间约 10⁶ 量级。老师傅凭经验在其中"盲调"，效果因人而异、早晚班波动 1~3% 良率、换线平均试错 2~4 小时、经验不可传承。

核心命题：把"调炉"从以人为主的经验手艺变成以数据+模型驱动的系统工程。系统先"学会"老师傅，再在稳定、全局寻优、可解释、自进化四项能力上系统性超越。

5. 特征工程

从 6 通道热电偶的原始曲线采样点（约 180~360 点/通道）压缩为 8 维特征，并拼接板/锡膏元信息。

5.1 曲线→特征计算（Python 实现）

import numpy as np

def extract_features(times, tc_signals):
    """
    times: array of sample timestamps (s)
    tc_signals: (6, N) array, 6通道热电偶温度
    返回: dict of 8 features
    """
    avg_curve = tc_signals.mean(axis=0)  # 6通道均值
    dt = np.gradient(times)
    dT = np.gradient(avg_curve)

    features = {
        'peak_temp':    float(avg_curve.max()),
        'tal':          float(np.sum(avg_curve >= 217) * dt.mean()),
        'ramp_up':      float(np.percentile(dT[:len(dT)//3], 90)),
        'ramp_down':    float(np.percentile(dT[len(dT)*2//3:], 10)),
        'delta_t':      float(tc_signals.max(axis=1).ptp()),
        'soak_temp':    float(avg_curve[len(avg_curve)//4:len(avg_curve)//2].mean()),
        'time_above_183': float(np.sum(avg_curve >= 183) * dt.mean()),
        'curve_duration': float(times[-1] - times[0])
    }
    return features

6. 模型层设计（三个核心模型）

模型1 · 工艺代理模型

功能：设定温度/链速/风量 → 预测炉温曲线（6 通道）

方法：物理热模型（传热方程）做基线 + 神经网络残差校正（Physical-Informed Neural Network, PINN）。小样本阶段物理模型兜底，数据多了 NN 逐渐主导。

训练数据：历史所有炉次的 (设定值, 实测曲线) 对。每个炉次提供 6×180 个标签点。

class SurrogateModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.physics_net = SimpleThermalSolver()  # 物理模型
        self.residual_nn = nn.Sequential(         # 残差网络
            nn.Linear(12, 64), nn.ReLU(),
            nn.Linear(64, 128), nn.ReLU(),
            nn.Linear(128, 6 * 180)       # 6通道×180时间点
        )
    def forward(self, x):
        base = self.physics_net(x)              # 物理预测
        residual = self.residual_nn(x)          # NN 校正
        return base + residual

理由：纯 NN 在小样本时可能预测物理上不可能的曲线（如峰值 400℃），物理约束保安全下限。

模型2 · 质量预测模型

功能：曲线特征 + 板/锡膏元信息 → 各缺陷类型的概率

方法：多标签 XGBoost（MultiOutputClassifier），每个缺陷类型一个二分类器。损失函数加权以应对样本不平衡（坏板仅 3%）。

from xgboost import XGBClassifier

模型3 · 参数推荐引擎

功能：新板 BOM + 锡膏 + 工艺窗口 → 最优温区设定 + 链速

方法：贝叶斯优化 (Bayesian Optimization) 在代理模型上搜。搜索空间: 10 个温区温度 + 链速，共 11 维。目标函数 = 质量预测模型输出的缺陷概率（取最小）。约束: peak∈[235,250], TAL∈[45,90], ramp_up∈[1,3] 等。每次评估在代理模型上推理（毫秒级），无需真跑炉子。

详见 §7。

7. 推荐引擎算法详解

from bayes_opt import BayesianOptimization
from bayes_opt.util import UtilityFunction
import numpy as np

class ReflowRecommender:
    def __init__(self, surrogate_model, quality_model, featurizer):
        self.surrogate  = surrogate_model     # 代理模型
        self.quality    = quality_model       # 质量预测模型
        self.featurizer = featurizer          # 特征提取器
        self.window     = ProcessWindow()     # 工艺窗口约束

    def objective(self, **params):
        """BO 目标函数: 最小化缺陷概率"""
        setting = {'zones': [params[f'z{i}'] for i in range(1,11)],
                   'speed': params['speed']}
        # 1. 代理模型预测曲线
        curve_6ch = self.surrogate.predict(setting)
        # 2. 提取曲线特征
        feats = self.featurizer(curve_6ch)
        # 3. 质量模型预测缺陷概率
        defect_probs = self.quality.predict_proba(feats)
        # 4. 检查工艺窗口约束（不满足则罚分）
        penalty = self.window.penalty(feats)
        return -(defect_probs.max() + penalty)  # BO 默认最大化

    def recommend(self, product, solder_paste, top_k=3):
        """对新板给出 top-k 个推荐方案"""
        pbounds = self._build_bounds(product, solder_paste)
        opt = BayesianOptimization(
            f=lambda **p: self.objective(**p),
            pbounds=pbounds,
            random_state=42
        )
        # 探索阶段: 随机采样 12 个初始点
        opt.maximize(init_points=12, n_iter=36)
        # 取 top-k, 并解释
        results = []
        for i, res in enumerate(opt.res[:top_k]):
            explanation = self._explain(res['params'], product)
            risk = self._assess_risk(feats)
            results.append({'setting': res['params'], 'score': -res['target'],
                           'explanation': explanation, 'risk': risk})
        return results
约束注入：内置工艺窗口硬约束：peak∈[235,250], TAL∈[45,90], ramp_up∈[1,3]。越界参数 100% 拦截，不进入推荐结果。异常时自动回退到该产品历史最优设定（安全兜底）。

8. 知识库 RAG 设计

架构：本地 RAG（检索增强生成）

不把工艺数据外传，全链路本地运行。

  1. 离线建库：收集 SOP 文档 / 老师傅经验 / 历史异常案例 / 行业规范 → 用本地 Embedding 模型（BGE-M3, 1024d）转为向量 → 存入 pgvector。
  2. 在线问答：用户提问 → 同模型 Embedding → pgvector 检索 Top-5 相关文档 → 拼 Prompt 送入 Qwen-7B（本地部署）→ 返回带引用来源的回答。
  3. 持续更新：每次反馈闭环标记的对/错案例自动入知识库（经人工确认后）。
# RAG 问答伪代码
def kb_answer(question):
    q_vec = embed(question)                               # BGE-M3 embedding
    docs  = pgvector.search(q_vec, top_k=5)             # 向量检索
    context = "\n---\n".join([d.content for d in docs])
    sources  = [d.source for d in docs]
    prompt = f"""你是回流焊工艺专家。请基于以下知识库内容回答问题。
只使用下面提供的信息，不要编造。如果信息不足，请说"根据现有资料无法确定"。

知识库内容:
{context}

问题: {question}

请给出建议并标注引用来源。"""
    answer = qwen.generate(prompt, max_tokens=512)    # 本地 Qwen-7B
    return {'answer': answer, 'sources': sources}

参数硬拦截推荐引擎输出的设定值在写入 PLC 前过三道校验：①是否在工艺窗口 (±3σ) ②与历史安全设定库比对 ③物理合理性（如升温段必须递增）。任一不通过 → 阻断 + 告警。
人确认 + 审计下发须通过审批流：工艺员在 Web 点"确认下发"→ 二次弹窗确认 → 写入

15. 系统如何超越老师傅（量化论证）

本技术方案不是要把老师傅"替换掉"，而是把老师傅好的那部分数据化、规模化，加上 AI 天然擅长的多维能力。以下每一行都有对应技术实现：

AI 优势技术实现量化预期
零漂移稳定模型推理无疲劳/情绪，同一输入 → 同一输出；早晚班无差异良率波动 σ 降低 ≥ 40%
全局寻优贝叶斯优化在 11 维空间搜 36 轮（≈ 10⁶ 等效评估），每次在代理模型推理（毫秒），人无法做到推荐良率 ≥ 老师傅 +1.5pp
全历史经验模型在 3.2 万炉次上训练，覆盖所有产品，不会"换料就抓瞎"新板首炉合格 → ≥ 92%
可解释可审计SHAP 归因 → "回流区+6℃是因为 BGA 热容大"；每次推荐可回溯工艺员信任度 ↑ 采纳率 ≥ 80%
设备老化自校正代理模型每月用最新曲线重校准，抵消加热器衰减无需人"将就"，始终最优
知识不流失所有经验入知识库 RAG；新人 2 周上岗而非 2 个月培训周期 ↓ 75%
数据飞轮每炉反馈回灌训练集 → 模型月月更准 → 采纳率↑ → 反馈更多6 个月后 AUC +0.05
人不可替代的部分：罕见新物料/新锡膏的首次调参、产线突发异常（如笼卡死、加热管断）、跨工序联调——这些仍由老师傅主导。系统定位是 "最强辅助"：把 80% 的常规调炉自动化，让人专注最难的 20%。
