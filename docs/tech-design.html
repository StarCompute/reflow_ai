1. 目标与范围

问题：回流焊炉温曲线由老师傅凭经验调温区/链速/风量，存在 难招、波动大、换线慢、经验随人走 四类痛点。

目标：用 数据 + AI 把"调炉能力"做成可复用系统——记录数据 → 建模预测缺陷 → 给新板推荐参数 → 人确认下发 → 持续迭代。试点阶段做到 "辅助推荐"（阶段③），不直接全自动。

范围边界：本期不改造炉体硬件；不直接自动改 PLC；知识库大模型本地部署，工艺机密不出厂。

4. 模型层设计（三个核心模型）

模型1 · 工艺代理模型（设定值 → 预测曲线）

输入 zone_temp / chain_speed / 风量 → 输出预测实际炉温曲线。用途：未真过炉即可预判曲线是否落在工艺窗口。
方法：物理热模型（传热方程）做基线 + 数据驱动校正（神经网络/GP）；小样本先用物理模型保底。

模型2 · 质量预测模型（曲线+板+锡膏 → 缺陷概率）

输入 6 个曲线特征 + 板厚/铜面积/BGA 有无/锡膏型号 → 输出 虚焊/桥连/立碑/空洞 概率。
方法：XGBoost / 随机森林（小样本友好、可解释）。这是替代老师傅"预判风险"的核心。

模型3 · 参数推荐引擎（新板 → 最优设定）

输入 新产品 BOM + 锡膏 + 工艺窗口约束 → 输出推荐 zone_temp / chain_speed。
方法：贝叶斯优化 / 遗传算法在代理模型上搜索；输出附带"风险提示"与"为什么这么设"（可解释）。

🤖 系统如何"超越"老师傅（设计意图）

三个模型协同，不只复刻经验，更补齐人做不到的四点：

  • 全局寻优：代理模型 + 贝叶斯优化在 10 温区×链速×风量 的全空间搜最优，人凭直觉只能调 1~2 个变量。
  • 零漂移稳定：同一标准 24h 不变，消除早晚班、疲劳带来的 1~3% 波动。
  • 可解释可归因：SHAP 给出"为什么这么设 / 哪里会虚焊"，比"手感"更可信、可审计。
  • 数据飞轮自进化：每炉数据回灌，模型越用越准；设备老化时由代理模型自校正，人经验到顶就封顶。

验收线（见 §9）：AI 推荐炉次平均良率 ≥ 同产品老师傅手调 +1.5pp，且良率波动（σ）降低 ≥40%。老师傅保留"人确认"兜底权，专注监督与异常处理。

5. 算法细节

5.1 特征工程（曲线 → 6 维特征）

特征含义无铅合格区间计算
peak_temp峰值温度235~250 ℃曲线最大值
TAL液相线(217℃)以上时间45~90 s≥217℃ 累计时长
ramp_up升温斜率1~3 ℃/s预热段斜率
ramp_down降温斜率-1~-4 ℃/s冷却段斜率
delta_T板面最大温差<15 ℃各通道峰值差
soak_temp恒温平台温度150~200 ℃恒温段均值

5.2 质量预测模型训练（伪代码）

# 输入 X: 曲线特征 + 板/锡膏元信息；y: 缺陷标签(0/1)
  X, y = load("reflow_2024H1")      # 3.2万炉次
  X_tr, X_te, y_tr, y_te = split(X, y, test_size=0.2, stratify=y)
  model = XGBClassifier(max_depth=6, n_estimators=300,
                        scale_pos_weight=class_weight)
  model.fit(X_tr, y_tr)
  auc  = roc_auc_score(y_te, model.predict_proba(X_te)[:,1])  # 目标 ≥0.85
  recall = recall_score(y_te, model.predict(X_te))             # 目标 ≥0.80
  # 可解释：输出特征重要性 + SHAP 单样本归因
  explain = shap.Explainer(model); shap_values = explain(X_te[:1])

# 在代理模型 f 上搜索使"预测缺陷概率最小"的设定

7. 技术栈选型

用途推荐说明
数据采集Python + OPC UA / Modbus从设备读数
存储MySQL + InfluxDB + pgvector关系/时序/向量各司其职
建模Python + scikit-learn / XGBoost / PyTorch主流、免费
后端FastAPI / Flask轻量 REST
前端Vue/React + ECharts看板与曲线可视化
知识库RAG + 开源大模型(本地, 如 Qwen)工艺机密不出厂
选型原则：采集与数据层自建（数据主权必须握在自己手里）→ 模型层先用开源 XGBoost 起步 → 知识库用开源大模型本地部署。仅"炉温仿真"类专业模块可考虑采购。

8. 安全与闭环

▸ 网络隔离：系统与产线 PLC 网络隔离，下发走审批接口，不直连乱改。

▸ 安全红线：任何越界参数（如峰值>260℃、TAL 超窗）100% 拦截并报警；异常时一键回退 ≤1 秒。

▸ 人确认：所有下发由工艺员点"采纳并下发"才生效，系统只建议。

▸ 反馈闭环：采集→AI推荐→人确认→过炉→检测→反馈→模型迭代（见 09 反馈章）。

10. 风险与对策

风险对策
数据不全/质量差先选数据全的试点线；X-Ray 未接则缺陷标签缺失，优先补采集
模型不可解释被拒用强制输出 SHAP 归因 + 风险提示，人始终确认
老师傅抵触定位为"辅助"非"取代"；采纳纳入正向激励
越界参数害事100% 硬拦截 + 一键回退，安全测试先行
算法人才依赖模型层用开源起步，文档化训练流水线
