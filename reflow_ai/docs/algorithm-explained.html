1. 这系统到底在干什么？

回流焊（Reflow）是电子厂给电路板"焊锡"的工序：把贴好零件的 PCB 板送进一条长长的高温炉子， 炉子分 10 个温区，每个区设一个温度，再配上一条传送链的链速（板子走多快）。 温度曲线像"先预热 → 再均热 → 冲到峰值熔化焊锡 → 冷却定型"。

曲线长得好不好，直接决定焊点有没有缺陷：温度不够会虚焊（没粘牢）， 太高会桥连（相邻脚短路），受热不均会立碑（零件翘起来）……

过去这些温区温度、链速全靠老师傅凭经验调。本系统用 AI 做三件事：

① 代理模型：输入温区设定 → 预测出炉温曲线 ② 质量模型：看曲线 → 预测 5 类缺陷概率/良率 ③ 推荐引擎：给新板 → 搜出最优温区+链速

2. 一个生活比喻：做蛋糕

把回流焊想成 用烤箱做蛋糕：

  • 温区设定 + 链速 = 你设定的"烤箱各段温度 + 烤多久"（这就是旋钮/参数）。
  • 出炉的炉温曲线 = 蛋糕在烤箱里"实际经历的温度变化"。
  • 缺陷（虚焊/桥连…） = 蛋糕"塌了 / 焦了 / 没熟"。
  • 代理模型 = 一个"烤箱模拟器"：你告诉它旋钮怎么拧，它算出蛋糕会经历什么温度。
  • 质量模型 = 一个"品控师傅"：看温度曲线，判断蛋糕会不会出问题、成功率多少。
  • 推荐引擎 = 一个"智能助手"：你说"我要做 8 寸芝士蛋糕"，它自动帮你把旋钮调到最稳的组合。

下面所有算法，本质都是围绕"旋钮 → 温度曲线 → 好不好"这条链路在转。

4. 三大算法逐个拆（小白版）

点下面的标签切换。每个算法都按"它像什么 → 解决啥 → 怎么做 → 代码在哪"四步讲。

像什么？

一个 "烤箱模拟器"。你告诉它 10 个温区设多少度、链速多少，它立刻算出板子会经历的一条温度曲线（6 个测温点各一条）。

解决啥问题？

真实炉子每次实测曲线要等板子真的过炉才知道，太慢。代理模型让你不用真烧一遍就能"脑补"出曲线，供后面判断良率、做安全校验用。

怎么做？（物理 + AI 残差 = PINN 思路）

它分两层，这是关键巧思：

  • 第一层·物理基线：用一条简单的"一阶滞后"公式，把温区设定平滑插值得目标曲线，再让每个测温点按自己的"热惯性"慢慢追上目标。 这保证了物理上说得通（不会算出反常识的曲线），且不需要数据就能跑。
  • 第二层·AI 残差：真实炉子和理想公式总有偏差。用神经网络（MLP）去学"理想曲线"和"真实曲线"之间的残差（差值）， 预测时 最终曲线 = 物理基线 + 神经网络修正。这就是 PINN（物理信息神经网络）的简化思路：让物理当"骨架"，AI 补"血肉"。
# models/surrogate.py（简化示意）
def predict(setting):
    target = 温区之间平滑插值          # 避免阶梯跳变造成假陡坡
    for 每个测温通道 c:
        ac = 基础滞后 × 通道差异 × 链速因子   # 不同位置热惯性不同→峰值有差
        ch = 一阶滞后追踪(target, ac)          # 物理骨架
        out[c] = ch + 通道偏移
    return out + 神经网络残差修正             # AI 补血肉

为什么这样稳？纯神经网络容易"胡说"出物理上不可能的曲线；纯物理公式又不够准。 两者相加，既准又不会离谱——这也是工业 AI 的常见套路。

像什么？

一个 "品控老师傅"。你把一条温度曲线（加上这块板的厚度、零件密度等信息）递给它， 它告诉你：虚焊 / 桥连 / 立碑 / 空洞 / 锡珠 这 5 种缺陷各自的概率，以及综合良率。

解决啥问题？

缺陷要等板子焊完、过 AOI/X 光检测才知。质量模型让你看曲线就能提前预判哪批板子危险，也作为推荐引擎的"打分器"。

怎么做？（多标签分类）

  • 输入（16 个数字）：8 个"曲线特征"（峰值温度、液相线以上时间、升温斜率…）+ 8 个"板子信息"（板厚、铜面积、BGA 数量、锡膏型号、环境温湿度…）。
  • 输出（5 个 0~1 的概率）：每个缺陷"发生的可能性"。良率 = 1 − 最大缺陷概率。
  • 算法：演示用随机森林（RandomForest）做多标签预测；生产可换 XGBoost。训练数据来自历史"曲线 + 实际检测结果"。
# 输入 16 维 = 8 曲线特征 + 8 板子/环境信息
X = [峰值, 液相线时间, 升温率, 降温率, 温差ΔT, 均热温,
     183℃以上时间, 曲线时长,
     板厚, 铜面积, BGA数, 最大BGA, 锡膏编码, 零件密度, 环境温度, 湿度]
# 输出 5 维概率
y = [虚焊P, 桥连P, 立碑P, 空洞P, 锡珠P]

演示数据的巧思：为了让模型"学得会、考得出来"，演示里缺陷标签不是乱编的， 而是由"曲线是否违反工艺窗口"推导——越界最严重的一项就是对应缺陷，全在窗口内就是"无"。 这样质量模型学到的就是"曲线→是否越界"的清晰边界，评估时 AUC 自然高。

像什么？

一个 "智能调参助手"。你说"这批是新板 PCB-A12、用 SAC305 锡膏"，它自动在 11 个旋钮 （10 温区 + 链速）里翻找，给出 Top-3 最稳的设定，并附上预测良率。

解决啥问题？

11 个旋钮组合数天文数字，人肉试错成本极高。推荐引擎用贝叶斯优化聪明地搜，几十次就找到好解； 而且任何违反安全窗口的解都会被 100% 挡掉。

怎么做？（贝叶斯优化 + 硬约束）

  • 代理模型当"预言机"：对每个候选设定，先让①代理模型算出曲线，再让②质量模型算出缺陷概率。
  • 目标函数：分数 = −(最大缺陷概率 + 1000 × 越界罚分)。 贝叶斯优化要"最大化分数"，所以它会拼命压低缺陷概率、并且绝不越界（罚分权重 1000 倍，越界代价极大）。
  • 搜素策略（GP + UCB）：先随机试十几次，用一个"高斯过程"模型去拟合"设定↔分数"的地形， 然后专挑"既可能高、又不确定"的点试（UCB 采集函数），越找越准。约 36 轮就收敛。
# models/recommender.py（核心目标函数）
def objective(**参数):
    curve = 代理模型.predict(参数)          # 脑补曲线
    feat  = 提取曲线特征(curve)
    proba = 质量模型.predict_proba(输入)     # 预判缺陷
    penalty = 工艺窗口.penalty(feat)        # 越界越多罚分越大
    return -(proba.max() + 1000.0 * penalty)   # 硬约束优先

为什么约束能满足率 100%？因为目标里越界罚分被放大了 1000 倍， 优化器一旦发现某设定越界，分数立刻暴跌，于是它会"绕着红灯走"，最终吐出的 Top-3 必然落在安全窗口内。

5. 数据质量红线（先体检，再训练）

AI 吃垃圾数据就会拉垃圾结论。训练前先用 data/validate.py 过 8 条红线，任意一条不达标就阻断训练：

红线含义不达标会怎样
R1run_id 唯一且非空无法追溯
R2每通道曲线采样点 ≥ 60曲线太稀，特征失真
R3实测峰值 − 设定峰值 ≤ 30℃传感器/设定异常报警
R4追溯率 100%（每炉次都有质量结果）样本无标签，学不会
R5end_time > start_time时间倒流，数据错
R6lot_id 非空（板级追溯）板子对不上
R7defect_type 在枚举内脏标签
R8oven_id / product_id / chain_speed 必填关键字段缺失

演示里这 8 条全部 OK，所以训练照常进行。

6. 验收指标（考卷怎么打分）

训练完用 training/evaluate.py 出三张"成绩单"，对照验收线：

质量模型 · macro-AUC

AUC 衡量"给一对好坏样本，模型能否分清"的能力，1.0 满分，0.5 等于瞎猜。

验收线 ≥ 0.85 演示实测 ≈ 0.99

对 5 类缺陷各算一个 AUC 再取平均（macro）。演示里 5 类都能算出来（每类都有正样本）。

代理模型 · 曲线 RMSE

预测曲线和真实曲线差多少℃（均方根误差），越小越准。

演示实测 ≈ 0.04℃

因为演示曲线本就由同一套物理公式生成，所以残差极小——这正好说明"物理骨架"立住了。

推荐引擎 · 约束满足率 + 平均预测良率

把推荐出的 Top-1 设定送回代理模型算曲线、查工艺窗口：100% 不能越界（验收线）。

约束满足率 演示 = 100% 平均预测良率 ≈ 0.79

满足率 100% 说明安全网关那道"红灯"真的拦得住；良率 0.79 表示推荐设定预计近八成板子一次过关。

7. 为什么代码里有"两套工艺窗口"？

process_window（生产规范）

来自技术文档 V2 §7 的真实行业标准：峰值∈[235,250]℃、升温 1~3℃/s……

用在 API 安全网关 /dispatch：任何要下发给真炉子的设定，先用代理模型算曲线、查这个窗口，越界 100% 拦截并写审计日志。

demo_window（演示窗口）

本仓库的玩具物理模型还造不出完全符合真实规范的曲线，所以单独设一套"它力所能及"的窗口。

演示造数（缺陷标签）和推荐引擎约束都用它，保证演示能自洽跑通。等接入真实数据和真实物理后，训练和推荐统一改用 process_window。

一句话：process_window 是"考卷标准答案"，demo_window 是"练习册难度"——先把练习做顺，再上真考卷。

9. 名词小词典

代理模型 / Surrogate

用"设定值"预测"出炉温度曲线"的模型。这里 = 物理公式 + 神经网络残差。

质量模型 / Quality

看曲线预判 5 类缺陷概率与良率的模型（多标签分类）。

推荐引擎 / Recommender

用贝叶斯优化在 11 维参数空间里搜最优温区+链速的模型。

贝叶斯优化 / Bayesian Optimization

一种"用尽量少的试错找到最优解"的方法：边试边建概率模型猜测地形，专挑有潜力的点试。

GP + UCB

高斯过程（Gaussian Process）当"地形预言机"，UCB（上置信界）当"下一步试哪"的策略。

AUC

衡量二分类能力的指标，0.5=瞎猜，1.0=完美。macro-AUC = 各类别 AUC 的平均。

PINN

物理信息神经网络：把物理规律当约束融进神经网络，既准又不胡说。本系统用"物理基线+残差"简化实现。

工艺窗口 / Process Window

一组"曲线特征必须落在的范围"，越界就意味着焊点高风险。是安全红线的量化版。

数据飞轮

推荐→下发→实际质量结果→回灌重训→模型更准→再推荐，越转越聪明的闭环。
